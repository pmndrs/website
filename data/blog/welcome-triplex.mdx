## What's Triplex?

If you've never heard of Triplex let me give you the run down. Imagine you're building a React Three Fiber app and you want to debug your scene. You add a debug camera, some lights, and definitely some helpers to see where the camera and lights are. And now you're just getting to the point where you can start actually placing things in your scene! So you alt tab between your browser and code trying to position it just right and... what a total time sink and a ton of code you need to maintain just to get started!

![](https://triplex.dev/_next/image?url=%2F_next%2Fstatic%2Fmedia%2Fvsce-codelens.a49ab9a4.png&w=3840&q=75)

Triplex lets you get rid of all that. Out of the box helpers, transform & input controls to visually update your components, and all changes can be saved straight back to code! Hell yeah. And it's not just limited to React Three Fiber, open your standard React components and use Triplex to visually edit them too! It's a game changer for building integrated 2D + 3D React apps.

The best part? It's free! If you still need convincing I'd recommend following the "[Your First 3D Component](https://triplex.dev/docs/get-started/starting-a-project/your-first-3d-component/vscode)" tutorial to learn the basics. It'll be fun I promise! And you don't need anything to get started.

![](https://triplex.dev/_next/image?url=%2Ftutorials%2Fvsce-3d%2F8.png&w=1920&q=75)

If you'd like to learn more about Triplex I [have a blog post there](https://triplex.dev/blog/triplex-moves-to-pmndrs) going over some of its history.
